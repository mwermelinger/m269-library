lib.bst module

A binary tree ordered by key.

class lib.bst.BinarySearchTree

Bases: object

Provide a collection of key-value pairs with unique and comparable keys.

A binary tree is either empty or consists of a node (called the root) and two subtrees (called the left and right subtree).

A Binary Search Tree (BST) is a binary tree in which - every node is a key-value pair, - every key in the left subtree is smaller than the root’s key, - every key in the right subtree is larger than the root’s key.

add(the_key, the_value)

Associate the_value to the_key in the tree. Return nothing.

Assume the_value is not None. If the tree has the_key, replace its associated value by the_value. If it hasn’t, add a new node with the_key and the_value.

height()

Return the height of the tree.

Follow the definition in M269: The height is the maximum number of edges from the root to any leaf. The height of the empty tree is -1.

in_order()

Do an in-order traversal of the tree.

Return a list of the keys in the tree, in the order visited.

is_empty()

Return True if the tree is empty, otherwise False.

post_order()

Do a post-order traversal of the tree.

Return a list of the keys in the tree, in the order visited.

pre_order()

Do a pre-order traversal of the tree.

Return a list of the keys in the tree, in the order visited.

remove(the_key)

Remove the node with the_key from the tree. Return nothing.

Do nothing if the tree hasn’t the_key.

smallest_key()

Return the smallest key in the tree.

Return None if the tree is empty.

value(the_key)

Return the value associated to the_key in the tree.

Return None if the tree hasn’t the_key.
